# Sprint index

Föreslagen plan: 9 sprintar. Varje sprint kan vara 1–2 veckor beroende på teamstorlek.

| Sprint | Fokus |
|---|---|
| Sprint 00 | Discovery, teknikval, arkitektur |
| Sprint 01 | Proxy skeleton och mock provider |
| Sprint 02 | Model registry och provider adapters |
| Sprint 03 | Feature extractor och JobDescriptor |
| Sprint 04 | Policy engine och route explanations |
| Sprint 05 | Routing scoring, fallback och dry-run |
| Sprint 06 | Logging, metrics och dashboard |
| Sprint 07 | Evals och outcome feedback |
| Sprint 08 | Security, multitenancy och beta hardening |

## Sprintprinciper

- Varje sprint ska avslutas med körbar demo.
- Latency ska mätas tidigt.
- Policies ska testas som kod.
- Varje ny route ska kunna förklaras.
